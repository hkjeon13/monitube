# Explore 소셜 갤러리 재디자인 계획

## 목적

`/explore`를 수집 결과를 단순히 나열하는 화면이 아니라, 채널과 동영상을 빠르게 훑고 다시 탐색할 수 있는 소셜 포스트형 라이브러리로 개선한다. 시각적 기준은 [Dribbble Social Posts](https://dribbble.com/shots/27195636-Social-Posts)의 넓은 여백, 강한 타이포그래피, 프로필 중심의 편집형 갤러리다.

레퍼런스의 원본 그래픽·브랜드·콘텐츠를 복제하지 않는다. Monitube의 모스 그린·오렌지 색상과 공개 YouTube 수집 데이터를 사용한다.

## 사용자 목표

- 지금까지 수집된 채널을 한눈에 발견한다.
- 최근 또는 반응이 높은 동영상을 썸네일 기반 카드로 탐색한다.
- 채널 카드에서 자동 갱신 핀 상태를 확인·변경한다.
- 영상 카드를 눌러 기존 상세/공개 댓글 패널로 진입한다.
- 모바일에서도 한 손으로 탐색하고 핀 상태를 변경한다.

## 화면 구조

### 상단 헤더

- `Explore` 제목과 짧은 설명을 크게 배치한다.
- 우측에는 저장된 결과만 다시 읽는 새로고침 버튼을 둔다. 이 버튼은 YouTube API 수집을 시작하지 않는다.
- 결과 수, 마지막 라이브러리 갱신 시각, 정렬 기준을 보조 정보로 표시한다.

### 핀 채널 스트립

- 핀된 채널을 일반 채널보다 먼저 노출한다.
- 카드에는 채널명, `@handle`, 수집 영상/댓글 수, 마지막 수집 시각, 다음 자동 확인 시각을 표시한다.
- 북마크 버튼은 `aria-pressed` 상태를 제공하며 핀/해제를 즉시 반영한다.
- 핀 설정 직후에는 수집 대상의 기존 canonical target에만 자동 작업을 연결한다. 같은 채널의 중복 작업은 만들지 않는다.

### 채널 디렉터리

- 공개 채널을 프로필 카드 형태로 표현한다.
- 첫 글자 기반의 임시 아바타 대신 채널 썸네일을 저장하게 되면 실제 채널 이미지를 사용한다.
- 카드 클릭은 해당 채널 필터가 적용된 동영상 갤러리로 전환한다.
- 채널별로 `영상 수`, `수집 댓글 수`, `최근 수집`을 표시한다.

### 동영상 포스트 갤러리

- YouTube 공개 썸네일을 `https://i.ytimg.com/vi/{youtube_video_id}/hqdefault.jpg` 형식으로 표시한다.
- 카드 비율은 세로·가로 혼합 masonry 느낌을 주되, CSS Grid에서 예측 가능한 높이를 유지한다.
- 썸네일 위에는 그라데이션을 적용하고 제목, 게시일, 조회수, 댓글 수를 겹쳐 표시한다.
- 카드 클릭은 기존 영상 상세 drawer를 연다.
- 빈 결과, 이미지 로드 실패, 댓글 미수집 상태를 명확한 대체 UI로 처리한다.

## 정보 구조와 라우팅

| 경로 | 역할 | Explore와의 연결 |
| --- | --- | --- |
| `/` | 선택 수집 대상 Overview | 핵심 분석 요약 |
| `/explore` | 전체 수집 라이브러리 | 채널·영상 탐색 및 핀 |
| `/sources` | canonical 수집 대상 관리 | 특정 대상 선택 |
| `/jobs` | 실행·쿼터 대기 상태 | 자동 핀 작업 상태 확인 |
| `/insights` | 댓글 주요 단어/분석 | 선택 대상의 분석 결과 |

Explore는 독립 경로를 유지한다. Overview 내부 앵커나 동일 화면의 숨김 섹션으로 되돌리지 않는다.

## 데이터 및 API

### 현재 재사용

- `GET /v1/explore`
  - 채널: 채널 ID, 핸들, 이름, 설명, 영상 수, 댓글 수, 마지막 수집 시각, target ID, 핀 상태
  - 동영상: YouTube 영상 ID, 제목, 설명, 게시일, 조회/좋아요/댓글 통계
- `PUT /v1/collection-targets/{targetId}/pin`
  - 핀 활성화 여부와 수집 간격을 갱신한다.
- 기존 `GET /v1/videos/{videoId}/comments`
  - 동영상 카드의 상세 drawer에서 저장된 댓글을 표시한다.

### 추가할 API 계약

`GET /v1/explore`에 아래 query parameter를 단계적으로 추가한다.

| 파라미터 | 용도 |
| --- | --- |
| `channelId` | 특정 채널의 동영상만 보기 |
| `sort=recent\|views\|comments` | 갤러리 정렬 |
| `cursor` / `limit` | 커서 기반 페이지네이션 |
| `pinnedOnly` | 핀 채널만 보기 |

응답에는 `nextCursor`와 `thumbnailUrl`을 포함한다. `thumbnailUrl`은 최초에는 영상 ID에서 API가 안전하게 조립하고, 추후 `videos.thumbnail_url` 저장 컬럼으로 전환할 수 있게 한다.

## 데이터 모델 확장

1. 1차 구현에서는 DB 마이그레이션 없이 `youtube_video_id`에서 썸네일 URL을 생성한다.
2. 채널/영상 썸네일을 영속화할 필요가 생기면 다음 nullable 컬럼을 추가한다.

```sql
ALTER TABLE channels ADD COLUMN thumbnail_url TEXT;
ALTER TABLE videos ADD COLUMN thumbnail_url TEXT;
```

3. YouTube `channels.list` 및 `videos.list`의 `snippet.thumbnails`에서 값을 upsert한다.
4. 썸네일 URL은 공개 메타데이터로만 취급하고, 만료·삭제 관리 정책은 기존 `expires_at`, `deleted_at`과 맞춘다.

## UI 구현 순서

1. Explore API에 정렬·필터·페이지네이션 계약을 추가하고 테스트한다.
2. `apps/web/app/explore/page.tsx`를 Explore 전용 컨테이너로 분리한다.
3. 핀 채널 스트립, 채널 필터, 정렬 드롭다운을 구현한다.
4. 썸네일 기반 반응형 갤러리와 이미지 fallback을 구현한다.
5. 카드에서 기존 영상 drawer를 열고, URL query로 필터/정렬 상태를 유지한다.
6. 핀 토글, 로딩 skeleton, 빈 상태, 오류 상태를 마무리한다.
7. 데스크톱·태블릿·모바일 화면을 시각 검수하고 운영 배포한다.

## 반응형 기준

| 화면 폭 | 채널 카드 | 동영상 갤러리 | 동작 |
| --- | --- | --- | --- |
| 75rem 초과 | 4열 | 4~5열 | hover로 메타데이터 강조 |
| 45~75rem | 2~3열 | 3열 | 카드 높이 축소 |
| 45rem 이하 | 2열 | 2열 | 터치 영역 2.75rem 이상, 텍스트 2줄 말줄임 |

`px` 고정 치수 대신 기존 기준처럼 `rem`, `clamp()`, CSS Grid의 `minmax()`를 사용한다.

## 접근성 및 성능

- 이미지에는 영상 제목 기반의 대체 텍스트를 제공한다.
- 핀 버튼은 현재 상태와 동작을 음성 안내한다.
- 카드 전체를 버튼 또는 링크 하나로 만들고 중첩 인터랙션을 피한다.
- `next/image` 또는 lazy loading으로 화면 밖 썸네일 요청을 지연한다.
- 갤러리는 초기 24개 이하만 렌더링하고 커서 기반 "더 보기"를 제공한다.
- 썸네일 실패 시 색상 그라데이션, 재생 아이콘, 영상 제목으로 구성된 fallback을 표시한다.

## 완료 기준

- `/explore`가 독립 페이지로 동작한다.
- 수집된 모든 채널/영상을 검색·필터·정렬할 수 있다.
- 동영상 카드에 썸네일 또는 동등한 fallback이 표시된다.
- 핀 상태가 새로고침 후에도 유지되고, 같은 대상의 활성 수집 작업은 하나만 존재한다.
- 모바일 360px 폭에서 카드가 넘치지 않고 모든 조작이 가능하다.
- API 테스트, 웹 프로덕션 빌드, 실제 배포 환경의 `/explore` 확인을 통과한다.
